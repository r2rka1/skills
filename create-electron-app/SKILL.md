# Create Electron App

Scaffolds a new Electron desktop application with sensible defaults and walks the user through initial configuration.

## Usage

```
/create-electron-app my-app
```

If no project name is provided, ask the user for one.

## Workflow

### 1. Validate Input

- [ ] Ensure a project name was provided. If not, ask the user.
- [ ] Confirm the target directory (`~/projects/<project-name>`) does not already exist.

### 2. Ask Preferences

Ask the user which options they want (defaults shown):

| Option | Default | Choices |
|--------|---------|---------|
| Renderer framework | React | React, Vue, Vanilla |
| TypeScript | Yes | Yes, No |
| Bundler | Vite | Vite, Webpack |
| Package manager | npm | npm, yarn, pnpm |

### 3. Create the Project

Use Electron Forge to scaffold the project:

```bash
npx create-electron-app@latest <project-name> --template=vite-typescript
```

Adjust the template based on user preferences:

| Preferences | Template |
|-------------|----------|
| Vite + TypeScript | `vite-typescript` |
| Vite + JavaScript | `vite` |
| Webpack + TypeScript | `webpack-typescript` |
| Webpack + JavaScript | `webpack` |

Run this from `~/projects/`.

### 4. Add Renderer Framework (if not Vanilla)

If the user selected React:

```bash
cd ~/projects/<project-name>
npm install react react-dom
npm install --save-dev @types/react @types/react-dom
```

Then update `src/renderer.ts` to `src/renderer.tsx` and create a basic React root:

```tsx
import React from 'react';
import { createRoot } from 'react-dom/client';

function App() {
  return <h1>Hello from Electron + React</h1>;
}

const root = createRoot(document.getElementById('root')!);
root.render(<App />);
```

Update `index.html` to include a `<div id="root"></div>`.

If the user selected Vue, install `vue` and set up a similar entry point.

### 5. Verify Setup

After creation, verify:

- [ ] `package.json` exists with `electron` dependency
- [ ] `forge.config.ts` (or `.js`) exists
- [ ] `src/main.ts` exists (main process entry)
- [ ] `src/preload.ts` exists
- [ ] `src/renderer.tsx` (or `.ts`) exists (renderer entry)
- [ ] App starts without errors: `npm start` (start and quickly stop)

### 6. Post-Setup Summary

Report to the user:

- Project location: `~/projects/<project-name>`
- Available commands:
  - `npm start` — launch the app in development mode
  - `npm run package` — package the app for the current platform
  - `npm run make` — create distributable installers
  - `npm run lint` — run linter
- Key files:
  - `src/main.ts` — main process (window creation, app lifecycle)
  - `src/preload.ts` — preload script (bridge between main and renderer)
  - `src/renderer.tsx` — renderer process (UI)
  - `forge.config.ts` — Electron Forge configuration
- Key concepts:
  - **Main process** — runs in Node.js, manages windows and system APIs
  - **Renderer process** — runs in Chromium, renders the UI
  - **Preload script** — securely exposes APIs from main to renderer via `contextBridge`

## Directory Structure

- `resources/` — persistent output and data files generated by this skill
- `scripts/` — reusable scripts for this skill's operations

## Script Management

When performing an operation that can be scripted:
1. Check `scripts/` for an existing script that handles this operation
2. If a script exists, execute it instead of doing the work inline
3. If no script exists and the operation is reusable, create one in `scripts/`, make it executable, then execute it
4. Reference any new scripts in this SKILL.md under "Available Scripts"
