# Create Project with Skills

Scaffolds a new project directory and sets it up with Claude Code skills by cloning the shared skills repository into its `.claude/` folder.

## Usage

```
/create-project-with-skills my-app
```

## Workflow

### 1. Validate Input

- [ ] Ensure a project name was provided as the argument. If not, ask the user for one.
- [ ] Validate the project name is a valid directory name (no spaces, special characters).
- [ ] Confirm the directory does not already exist in the current working directory.

### 2. Create Project Directory

```bash
mkdir -p <project-name>
```

### 3. Create `.claude` Directory and Clone Skills

```bash
# Create the .claude directory
mkdir -p <project-name>/.claude

# Clone the skills repo directly into .claude/skills
git clone https://github.com/r2rka1/skills.git <project-name>/.claude/skills

# Remove the .git directory so it's not a nested repo
rm -rf <project-name>/.claude/skills/.git
```

### 4. Verify and Report

- [ ] Confirm the project directory was created
- [ ] List all skills that were installed into `.claude/skills/`
- [ ] Show the resulting directory structure:

```
<project-name>/
  .claude/
    skills/
      <skill-1>/
        SKILL.md
      <skill-2>/
        SKILL.md
      ...
```

- [ ] Inform the user their project is ready and skills are available

## Notes

- The skills repository is: `https://github.com/r2rka1/skills`
- If the target directory already exists, ask the user whether to proceed or abort
- The `.git` directory from the cloned repo is removed to avoid nested git issues
- The user can initialize their own git repo in the project afterwards with `git init`

## Directory Structure

- `resources/` — persistent output and data files generated by this skill
- `scripts/` — reusable scripts for this skill's operations

## Script Management

When performing an operation that can be scripted:
1. Check `scripts/` for an existing script that handles this operation
2. If a script exists, execute it instead of doing the work inline
3. If no script exists and the operation is reusable, create one in `scripts/`, make it executable, then execute it
4. Reference any new scripts in this SKILL.md under "Available Scripts"
