## Usage

```
/create-vue-app my-app
```

The argument is the **project name** (required). It must be lowercase letters, numbers, and hyphens only.

## Prerequisites

- Node.js `^20.19.0 || >=22.12.0`
- npm, pnpm, yarn, or bun

## Workflow

Follow these steps in order:

### 1. Validate Input

- [ ] Ensure a project name was provided as an argument. If not, ask the user.
- [ ] Validate the name: lowercase letters, numbers, hyphens only. No leading/trailing hyphens.
- [ ] Determine the target directory. Default: current working directory. If the user ran `/setup-project` first, use that project's root.
- [ ] Confirm the target directory does not already contain a Vue project (no `vite.config.ts` or `vue.config.js`).

### 2. Detect Package Manager

Check which package manager is available, in order of preference:

1. If `pnpm` is installed, use pnpm
2. If `yarn` is installed, use yarn
3. Fall back to `npm`

Store the detected package manager for all subsequent commands.

### 3. Scaffold the Vue Project

Run `create-vue` non-interactively with recommended defaults:

```bash
# Using npm as example — substitute detected package manager
npm create vue@latest <project-name> -- \
  --typescript \
  --router \
  --pinia \
  --eslint-with-prettier \
  --no-vitest \
  --no-e2e \
  --no-jsx
```

If non-interactive flags are not supported by the current `create-vue` version, fall back to running interactively and selecting these options:

| Prompt | Selection |
|--------|-----------|
| Add TypeScript? | Yes |
| Add JSX Support? | No |
| Add Vue Router? | Yes |
| Add Pinia? | Yes |
| Add Vitest? | No |
| Add E2E Testing? | No |
| Add ESLint? | Yes |
| Add Prettier? | Yes |

### 4. Install Dependencies

```bash
cd <project-name>
<package-manager> install
```

### 5. Verify Project Structure

Confirm these key files/directories exist:

- [ ] `package.json` — contains `vue`, `vue-router`, `pinia` dependencies
- [ ] `vite.config.ts` — Vite configuration with Vue plugin
- [ ] `tsconfig.json` — TypeScript configuration
- [ ] `src/main.ts` — application entry point
- [ ] `src/App.vue` — root component
- [ ] `src/router/index.ts` — Vue Router setup
- [ ] `src/stores/` — Pinia stores directory
- [ ] `.eslintrc.cjs` or `eslint.config.js` — ESLint configuration
- [ ] `.prettierrc.json` — Prettier configuration

### 6. Verify Dev Server Starts

```bash
<package-manager> run dev -- --port 5173 &
# Wait a few seconds, then check if the process is running
# Kill the background process after verification
```

If the dev server fails to start, investigate and fix the issue before proceeding.

### 7. Report Results

Display a summary:

```
Vue 3 app created successfully!

  Project:         <project-name>
  Path:            <full-path>
  Package Manager: <detected-pm>
  TypeScript:      Yes
  Vue Router:      Yes
  Pinia:           Yes
  ESLint:          Yes
  Prettier:        Yes

  Commands:
    cd <project-name>
    <pm> run dev      — Start dev server (http://localhost:5173)
    <pm> run build    — Build for production
    <pm> run preview  — Preview production build
    <pm> run lint     — Lint and fix files
```

## Key Files Reference

| File | Purpose |
|------|---------|
| `vite.config.ts` | Build tool config, plugins, aliases |
| `src/main.ts` | App bootstrap — creates app, installs router & pinia |
| `src/App.vue` | Root component with `<RouterView />` |
| `src/router/index.ts` | Route definitions and router instance |
| `src/stores/` | Pinia store modules |
| `tsconfig.json` | TypeScript compiler options |
| `env.d.ts` | Type declarations for Vite env variables |

## Directory Structure
