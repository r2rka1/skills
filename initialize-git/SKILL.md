# Initialize Git

Sets up git and GitHub for the current project folder: creates a `.gitignore`, initializes a local repo, and creates a remote GitHub repository named after the folder.

## Usage

```
/initialize-git
```

Run this from the root of a project directory.

## Workflow

### 1. Verify Prerequisites

- [ ] Confirm the current directory is a project folder (not already a git repo)
- [ ] Confirm `gh` CLI is installed and authenticated (`gh auth status`)
- [ ] If already a git repo, ask the user if they want to proceed with just the GitHub remote setup

### 2. Create `.gitignore`

Create a `.gitignore` file with common defaults:

```gitignore
# Dependencies
node_modules/
vendor/
.venv/
venv/
__pycache__/

# Build output
dist/
build/
out/
*.o
*.pyc

# Environment and secrets
.env
.env.local
.env.*.local
*.pem

# IDE and editor
.vscode/
.idea/
*.swp
*.swo
*~

# OS files
.DS_Store
Thumbs.db

# Claude Code
.claude/

# Logs
*.log
npm-debug.log*
```

- [ ] If a `.gitignore` already exists, append only missing entries rather than overwriting

### 3. Initialize Local Repository

```bash
git init
git add .
git commit -m "Initial commit"
```

### 4. Create GitHub Repository

Use the current folder name as the repository name:

```bash
# Get folder name
REPO_NAME=$(basename "$(pwd)")

# Create GitHub repo (private by default)
gh repo create "$REPO_NAME" --private --source=. --push
```

- [ ] Ask the user if the repo should be **public** or **private** (default: private)

### 5. Verify and Report

- [ ] Confirm `.gitignore` was created/updated
- [ ] Confirm git repo was initialized
- [ ] Confirm GitHub repo was created
- [ ] Show the GitHub repo URL

## Notes

- The GitHub repository is created as **private** by default
- The repo name is derived from the current directory name
- If `gh` is not authenticated, guide the user to run `gh auth login`
- `.claude/` is included in `.gitignore` to keep skills and config local
