## Usage

```
/setup-project my-new-project
```

The argument is the **repository name** (required). It must be lowercase letters, numbers, and hyphens only.

## Workflow

Follow these steps in order:

### 1. Validate Input

- [ ] Ensure a repository name was provided as an argument. If not, ask the user.
- [ ] Validate the name: lowercase letters, numbers, hyphens only. No leading/trailing hyphens.
- [ ] Check that `/home/r2rka/projects/<repo-name>` does not already exist.

### 2. Create the Project Folder

```bash
mkdir -p /home/r2rka/projects/<repo-name>
```

### 3. Initialize Git Repository

```bash
cd /home/r2rka/projects/<repo-name>
git init
```

### 4. Copy Shared .claude Configuration

Copy the `.claude` folder from the parent projects directory, **excluding** skills research resources and Zone.Identifier files:

```bash
# Copy settings and folder structure
mkdir -p /home/r2rka/projects/<repo-name>/.claude/agents
cp /home/r2rka/projects/.claude/settings.json /home/r2rka/projects/<repo-name>/.claude/settings.json
cp /home/r2rka/projects/.claude/settings.local.json /home/r2rka/projects/<repo-name>/.claude/settings.local.json

# Copy all skills (structure + SKILL.md files + scripts)
cp -r /home/r2rka/projects/.claude/skills /home/r2rka/projects/<repo-name>/.claude/skills

# Remove Zone.Identifier files (WSL artifacts)
find /home/r2rka/projects/<repo-name>/.claude -name "*.Zone.Identifier" -delete
find /home/r2rka/projects/<repo-name>/.claude -name "*:Zone.Identifier" -delete
```

### 5. Create README.md

Create a `README.md` at the project root:

```markdown
# <repo-name>

> TODO: Add project description

## Getting Started

TODO: Add setup instructions

## Development

TODO: Add development guidelines
```

### 6. Create CHANGELOG.md

Create a `CHANGELOG.md` at the project root following Keep a Changelog format:

```markdown
# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [Unreleased]

### Added
- Initial project setup with README and .claude configuration
```

### 7. Create CLAUDE.md

Create a `CLAUDE.md` at the project root with minimal starter content:

```markdown
# CLAUDE.md

## Project Overview

TODO: Describe this project

## Tech Stack

TODO: List technologies used

## Key Commands

TODO: Add common commands (build, test, lint, etc.)
```

### 8. Create GitHub Repository

Create a **private** repository in the CreepteedLTD organization:

```bash
gh repo create CreepteedLTD/<repo-name> --private --source=/home/r2rka/projects/<repo-name> --push
```

If `--source` with `--push` doesn't work, fall back to:

```bash
gh repo create CreepteedLTD/<repo-name> --private
cd /home/r2rka/projects/<repo-name>
git remote add origin git@github.com:CreepteedLTD/<repo-name>.git
git add -A
git commit -m "Initial commit: project scaffold with README, CHANGELOG, and .claude config"
git branch -M main
git push -u origin main
```

### 9. Verify

- [ ] Confirm the repo exists: `gh repo view CreepteedLTD/<repo-name>`
- [ ] Confirm the local folder is set up: `ls -la /home/r2rka/projects/<repo-name>`
- [ ] Report success with the repository URL

## Output

After completion, display:
- Repository URL: `https://github.com/CreepteedLTD/<repo-name>`
- Local path: `/home/r2rka/projects/<repo-name>`
- Files created: README.md, CHANGELOG.md, CLAUDE.md, .claude/

## Directory Structure

- `resources/` — persistent output and data files generated by this skill
- `scripts/` — reusable scripts for this skill's operations

## Script Management

When performing an operation that can be scripted:
1. Check `scripts/` for an existing script that handles this operation
2. If a script exists, execute it instead of doing the work inline
3. If no script exists and the operation is reusable, create one in `scripts/`, make it executable, then execute it
4. Reference any new scripts in this SKILL.md under "Available Scripts"
