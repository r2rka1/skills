# Setup Skills

Creates the `.claude/skills/` directory in a project and populates it with skills from [r2rka1/skills](https://github.com/r2rka1/skills).

## Usage

```
/setup-skills
/setup-skills /path/to/project
```

If no path is provided, uses the current working directory.

## Workflow

### 1. Determine Target Project

- [ ] Use the provided path argument, or default to the current working directory
- [ ] Confirm the target directory exists and is a valid project (has a git repo or package files)

### 2. Create `.claude` Directory

```bash
mkdir -p <target-project>/.claude/skills
```

### 3. Clone Skills Repository

Clone the shared skills repo into a temporary location and copy skills into the project:

```bash
# Clone to temp directory
TEMP_DIR=$(mktemp -d)
git clone --depth 1 https://github.com/r2rka1/skills.git "$TEMP_DIR"

# Copy each skill directory (excluding repo-level files)
for skill_dir in "$TEMP_DIR"/*/; do
  skill_name=$(basename "$skill_dir")
  # Skip hidden directories and non-skill directories
  if [[ -f "$skill_dir/SKILL.md" ]]; then
    cp -r "$skill_dir" "<target-project>/.claude/skills/$skill_name"
  fi
done

# Clean up
rm -rf "$TEMP_DIR"
```

### 4. Verify and Report

- [ ] List all skills that were installed
- [ ] Show the resulting `.claude/skills/` directory structure
- [ ] Inform the user that skills are now available for use in their project

## Notes

- This does **not** modify any existing `.claude/skills/` content — it copies new skills alongside existing ones
- If a skill with the same name already exists, ask the user whether to overwrite or skip
- The skills repository is: `https://github.com/r2rka1/skills`

## Directory Structure

- `resources/` — persistent output and data files generated by this skill
- `scripts/` — reusable scripts for this skill's operations

## Script Management

When performing an operation that can be scripted:
1. Check `scripts/` for an existing script that handles this operation
2. If a script exists, execute it instead of doing the work inline
3. If no script exists and the operation is reusable, create one in `scripts/`, make it executable, then execute it
4. Reference any new scripts in this SKILL.md under "Available Scripts"
