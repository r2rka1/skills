# Update Skills

Pulls the latest skills from [r2rka1/skills](https://github.com/r2rka1/skills) main branch and updates the project's `.claude/skills/` directory.

## Usage

```
/update-skills
```

Run this from within a project that already has `.claude/skills/` set up.

## Workflow

### 1. Validate Project

- [ ] Confirm the current working directory has a `.claude/skills/` directory
- [ ] If not found, inform the user and suggest running `/setup-skills` first

### 2. Pull Latest Skills

Clone the skills repository at the latest main branch and update local skills:

```bash
# Clone to temp directory
TEMP_DIR=$(mktemp -d)
git clone --depth 1 --branch main https://github.com/r2rka1/skills.git "$TEMP_DIR"

# Update each skill from the repository
for skill_dir in "$TEMP_DIR"/*/; do
  skill_name=$(basename "$skill_dir")
  if [[ -f "$skill_dir/SKILL.md" ]]; then
    # Remove old version and copy new one
    rm -rf ".claude/skills/$skill_name"
    cp -r "$skill_dir" ".claude/skills/$skill_name"
  fi
done

# Clean up
rm -rf "$TEMP_DIR"
```

### 3. Preserve Local Skills

- [ ] Do **not** delete skills in `.claude/skills/` that are not present in the repository — these are local-only skills the user created
- [ ] Only overwrite skills whose directory names match skills in the repository

### 4. Verify and Report

- [ ] List all skills that were updated (with a count)
- [ ] Note any new skills that were added (not previously present locally)
- [ ] Note any local-only skills that were preserved (not in the repository)
- [ ] Show the resulting `.claude/skills/` directory listing

## Notes

- The skills repository is: `https://github.com/r2rka1/skills`
- Always pulls from the `main` branch to ensure stable versions
- Local `resources/` content inside skills is overwritten — these directories are expected to be empty in the repository

## Directory Structure

- `resources/` — persistent output and data files generated by this skill
- `scripts/` — reusable scripts for this skill's operations

## Script Management

When performing an operation that can be scripted:
1. Check `scripts/` for an existing script that handles this operation
2. If a script exists, execute it instead of doing the work inline
3. If no script exists and the operation is reusable, create one in `scripts/`, make it executable, then execute it
4. Reference any new scripts in this SKILL.md under "Available Scripts"
